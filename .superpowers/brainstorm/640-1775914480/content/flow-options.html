Where do the showcase pages live?

The Collective and Cortex showcase pages — where do they appear in the user's journey?

A

Inside the setup wizard

After the user selects Collective/Cortex as goals in Step 1, the wizard shows a dedicated showcase step for each selected product — full hero, lifecycle diagram, feature tags — before continuing to workspace confirmation and guardrails.

/setup?step=goals → pick Collective + Cortex
→ /setup?step=collective-intro
→ /setup?step=cortex-intro
→ /setup?step=access
→ /setup?step=guardrails
→ /setup?step=ready
B

First-visit gate at /collective and /cortex

After setup is complete, when the user navigates to /collective or /cortex for the first time, they see the showcase page with an "Activate →" CTA. Once activated, future visits go straight to the dashboard.

setup completes → /control
user clicks Collective in sidebar
→ /collective (showcase — first visit)
→ clicks "Activate Collective →"
→ /collective (dashboard — all future visits)
C

Dedicated intro routes, always accessible

The showcase pages live at permanent routes like /collective/intro and /cortex/intro. They're linked from the sidebar and from the control overview. The dashboard is at /collective (unchanged). Users can always return to the intro.

sidebar: Collective → /collective/intro (first) or /collective
/collective/intro — showcase, always accessible
/collective — dashboard (unchanged)
/cortex/intro — showcase
/cortex — dashboard (new)
