Keon Cortex  ·  Deterministic Memory Substrate

Vector databases
are not memory.

Cortex is a deterministic, outbox-driven memory engine built for autonomous multi-tenant AI systems. Every shard has a canonical SHA-256 identity. Ingestion is idempotent. Tenant isolation is fail-closed — no cross-tenant fallbacks, ever. The vector index is derivative, never primary.

Invariant I1
Deterministic Shard Identity
Same inputs always produce the same shard ID. No randomness. No timestamps. Memory is canonical.
Invariant I2
Fail-Closed Tenancy
Missing, empty, or mismatched tenant IDs raise immediately. No silent pass-through. No cross-tenant bleed.
Invariant I3
Idempotent Ingestion
The same event cannot create duplicate shards. Outbox-driven processing guarantees no lost writes, no phantom duplicates.
Invariant I4
Replayable Processing
Exclusive leases. Atomic processing. Expired leases recover automatically. The system heals itself.
The four properties of governed execution
Every action. Every property. Enforced.

Policy-bound. Receipt-verified. Append-only. Fail-closed. These aren't aspirations — they're architectural invariants. Violations are bugs. There are no exceptions.

Policy-bound
Every action is evaluated against policy before it executes. No exceptions.
Receipt-verified
Every decision is cryptographically signed and cannot be altered after issuance.
Append-only
The ledger is immutable. Events are sequenced and sealed against reordering or deletion.
Fail-closed
On error or ambiguity, Keon denies — not permits. Safety is the default state.
